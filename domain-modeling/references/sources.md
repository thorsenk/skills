# Source Grounding

Use these sources for DDD methodology and attribution. They are not subject-matter sources for the user's domain. Load them when methodology needs support; research relevant primary sources separately when the model depends on missing subject knowledge.

## Primary DDD vocabulary

- Eric Evans, *Domain-Driven Design: Tackling Complexity in the Heart of Software*
- Eric Evans, *Domain-Driven Design Reference: Definitions and Pattern Summaries*
  - Source: https://www.domainlanguage.com/wp-content/uploads/2016/05/DDD_Reference_2015-03.pdf
  - Use for: domain, model, ubiquitous language, bounded context, context map, core domain, aggregate, entity, value object, and domain event.

## Practical modeling workflows

- ddd-crew, Domain-Driven Design Starter Modelling Process
  - Source: https://github.com/ddd-crew/ddd-starter-modelling-process
  - Use for: greenfield kickoff, current-state assessment, discovery, decomposition, strategy, context definition, and iterative modeling.

- ddd-crew, Bounded Context Canvas
  - Source: https://github.com/ddd-crew/bounded-context-canvas
  - Use for: purpose, responsibilities, inbound and outbound communication, collaborators, business decisions, assumptions, verification metrics, and open questions.

- ddd-crew, Context Mapping
  - Source: https://github.com/ddd-crew/context-mapping
  - Use for: relationships between contexts, systems, teams, upstream and downstream dependencies, published language, and adapter boundaries.

- ddd-crew, Core Domain Charts
  - Source: https://github.com/ddd-crew/core-domain-charts
  - Use for: strategic importance, differentiation, complexity, migration priority, and core, supporting, or generic classification.

- ddd-crew, Aggregate Design Canvas
  - Source: https://github.com/ddd-crew/aggregate-design-canvas
  - Use for: invariants, lifecycle, consistency boundaries, domain events, and aggregate responsibilities after context boundaries are clear.

## Agent workflow inspiration

- Matt Pocock, Skills for Real Engineers
  - Source: https://github.com/mattpocock/skills
  - Use for: agent-oriented shared language, exploration loops, `CONTEXT.md`, and ADR-style documentation.
  - Treat Evans and ddd-crew as the preferred sources for DDD methodology.

## Attribution rule

Cite local project files or stakeholder statements for project-specific claims. Cite relevant external sources for subject-matter claims and the sources above for methodology claims.
