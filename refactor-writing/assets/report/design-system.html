Patterns

Output Deviations

Recognizable system failures

These examples diagnose what the system prevents. They are not alternate visual styles.

RW-01

Grid drift

Related headings, findings, and bodies begin on competing horizontal anchors.

RW-02

Evidence separation

A proposal or rationale is detached from the exact language it supports.

RW-03

Viewport overflow

A page or component requires horizontal scrolling at a supported width.

RW-04

Color-only meaning

Status, navigation, or changed language cannot be understood without color.

RW-05

Motion lockout

Essential content stays hidden when JavaScript, observation, or motion fails.

RW-06

Copy dependency

The final document cannot be read or selected when copy behavior fails.

RW-07

Portability break

The artifact requires a network resource or a file outside its generated folder.

RW-08

Token fork

A raw value or duplicate layout rule compensates for a weak shared role.

RW-09

Motion excess

Looping, parallax, blur, scale, or simultaneous effects compete with reviewed language.

RW-10

Affordance substitution

Spotlight, hover, or color replaces the base border, focus ring, text label, or semantic state.

RW-11

Input-exclusive effect

A pointer-only effect carries meaning with no keyboard or static equivalent.

RW-12

Effect leakage

Decorative or interactive effects remain in print, forced colors, no-JavaScript, or unsupported-mask states.

RW-13

Mode drift

A component bypasses semantic roles, changes anatomy, or loses required contrast in one color mode.

RW-14

Icon ambiguity

An unsupported, inconsistent, or unlabeled icon carries action or status meaning without visible text.

RW-15

Catalog fiction

Documentation presents an imitation, unsupported state, or incomplete specimen as production capability.

RW-16

Material shortcut

A procedural or scale-dependent visual phenomenon is replaced with a stretched raster or misleading cosmetic substitute.

How it works

Evidence boundaries, ordered method, redactions, assumptions, and artifact structure.

System

This live teaching catalog: tokens, primitives, effects, components, patterns, states, deviations, and QA.

Dependencies

No frameworks, external fonts, icon packages, production network requests, or absolute navigation.

Portable assets

artifact.css, artifact.js, and four local HTML pages move together. The Hero particle field is generated as inline SVG at runtime.

Enhancement boundary

Primary content, component borders, static semantics, and selection work without JavaScript.

Catalog-only controls

Reveal and response-token replay exist only to inspect the system; generated report pages never include them.

QA contract

Prove the profile, not just the catalog

Release evidence must cover materially different content, interaction modes, responsive widths, enhancement failure, and both report branches.

Content fixtures

Recommendation report, complete no-change report, long SKILL.md source, and process explanation.

Viewports

Desktop, tablet, and phone with zero page-level or component-level horizontal overflow.

Interaction

Relative navigation, three modes, copy success/fallback, reveal, replay, spotlight, keyboard order, and visible focus.

Resilience

Reduced motion, touch, no JavaScript, forced colors, print, long content, missing mask support, and clear console.

Current evidence date

2026-07-23. Refresh after any token, component anatomy, effect, or profile change.

Release checks

Run the renderer fixtures, node scripts/validate-skills.mjs, browser verification, and git diff --check.
