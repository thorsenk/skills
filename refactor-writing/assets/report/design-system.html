States and resilience

Meaning survives enhancement failure

Labels, structure, and readable source carry the message before color, motion, or copy behavior is applied.

No change

Use the explicit heading “No change recommended” and return unchanged copy.

Long content

Headings, paths, source text, and unbroken strings wrap without widening the page.

No JavaScript

All content stays visible and selectable; only motion and copy assistance are absent.

Reduced motion

Reveal transitions collapse to effectively zero duration.

Print

Controls disappear, paper becomes white, and component boundaries remain visible.

Keyboard

Links, buttons, and selectable documents expose a visible focus ring.

Output Deviations

Recognizable failures

These examples name what the system prevents. They are diagnostic rules, not alternate styles.

RW-01

Grid drift

Related headings, findings, and bodies begin on competing horizontal anchors.

RW-02

Evidence separation

A proposal or rationale is detached from the exact language it supports.

RW-03

Viewport overflow

A page or component requires horizontal scrolling at a supported width.

RW-04

Color-only meaning

Status, navigation, or changed language cannot be understood without color.

RW-05

Motion lockout

Essential content stays hidden when JavaScript, observation, or motion fails.

RW-08

Token fork

A one-off raw value or duplicated grid compensates for a weak shared rule.

Implementation

One profile, one adapter

The renderer supplies semantic HTML; this stylesheet maps the system roles; JavaScript adds optional copy and motion behavior.

Dependency boundary

No external fonts, frameworks, images, or production network requests.

Portability

Every runtime link uses a same-folder relative path beginning with ./.

Responsive checks

Inspect desktop, tablet, and phone, including long content and both report branches.

Accessibility target

WCAG 2.2 AA with semantic structure, keyboard access, visible focus, reduced motion, and non-color-only meaning.
