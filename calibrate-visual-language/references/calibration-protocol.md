## Phase outputs

### Frame

Record artifact types, target surfaces, use context, desired qualities, legibility and fidelity needs, unacceptable traits, generation environment, and explicit unknowns.

### Inventory

Create a reference matrix. Record each image role, which traits it applies to, which traits it does not apply to, provenance or rights notes when known, surface limits, and conflicts.

### Provisional grammar

Create rules only for traits supported by observation or an explicit decision. Label inferences. Rank unresolved variables by their effect on later decisions.

### Controlled specimens

Use stable fixture content. Compare two to four variants. Hold the following fixed unless one is the declared variable: information hierarchy, copy, aspect ratio, object count, major geometry, and output scale.

For physical-media calibration, hold substrate, tool family, scale, lighting or scan settings, and crop fixed. Change one stroke variable at a time. Use close-up photographs or scans of real specimens; a generated imitation can serve only as a rejected comparison.

### Confirmation

Test every claimed surface class and at least one pressure case. A desktop lock does not prove mobile behavior. A spacious fixture does not prove density limits.

### Approval

Record the approving person or delegated evaluator, date, specimen ID, approved traits, rejected traits, and open limitations. Approval is trait-specific.

## Decision ledger

Keep three live lists:

- `decisions`: accepted values with evidence and scope;
- `conflicts`: competing evidence or instructions that need resolution;
- `uncertainties`: plausible but unconfirmed inferences.

Each entry needs an ID, status, date, statement, evidence IDs, affected surfaces, and next action. A conflict remains blocking when it affects a claimed invariant.

## Controlled specimens

Each specimen record includes:

- stable ID;
- fixture ID;
- output path;
- fixed parameters;
- varied family and values;
- generation method and relevant conditions;
- observed strengths and failures;
- decision status.

If several variables must move together to create a coherent direction, first show the combined direction, then run targeted follow-ups for any trait that remains ambiguous.

## Minimum sequence

Resolve medium, palette, mark-making, geometry, spacing, density, annotation, typography, depth, primary surface, alternate surface, and boundary behavior when they are relevant. Skip irrelevant families explicitly instead of inventing rules.

## Revision path

Preserve the released package. Create a candidate revision with affected rules, affected locks, targeted fixtures, and at least one regression fixture. Inherit unchanged rules and references. Reapprove only the changed trait families, then repeat structural validation and forward tests.
