# Visual rule taxonomy

Use only families relevant to the intended artifacts.

| Family | Typical variables |
|---|---|
| Purpose | artifacts, audience, contexts, exclusions |
| Medium | physical or digital medium, substrate, rendering method |
| Palette | hue, value, saturation, contrast, accents |
| Mark-making | line weight, pressure, texture, confidence, imperfection |
| Form | geometry, radius, proportions, silhouette, edges |
| Composition | grid, focal path, hierarchy, balance, cropping |
| Spacing | protected perimeter, density, rhythm, grouping |
| Depth | shadow, overlap, perspective, translucency, material |
| Typography | family, scale, case, weight, alignment, legibility |
| Annotation | mark style, placement, connectors, density, voice |
| Imagery | subject treatment, anatomy, environment, detail |
| Fidelity | sketch-to-polish level, realism, deliberate incompleteness |
| Motion | timing, easing, continuity, transitions |
| Output | aspect ratio, resolution, background, file constraints |

Each rule records:

- ID, family, and plain-language statement;
- variability: `invariant`, `controlled`, or `flexible`;
- status: `open`, `provisional`, `approved`, `rejected`, or `superseded`;
- evidence reference IDs and confidence;
- permitted range when applicable;
- applicable surfaces and surface-specific exceptions.

An invariant must trace to an approved decision or approved lock. A controlled rule needs an explicit range. A flexible rule may adapt to content but cannot break an invariant.
