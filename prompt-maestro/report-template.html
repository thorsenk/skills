Findings

What blocks readiness?

Findings are ranked by execution risk and rework potential, not wording preference.

Search scope is too broad

High-12 pts
Evidence

Broad local-machine and sibling-folder language can pull in unrelated private material.

Why it matters

The executor could waste time in irrelevant folders or surface material outside the intended review boundary.

Exact fix

Require explicit allowed roots and a stop rule before any local discovery begins.

No search receipt

High-8 pts
Evidence

The prompt requires sourced claims, but not roots searched, skipped roots, methods, or coverage limits.

Why it matters

A later reader cannot tell whether the review was broad enough or whether relevant paths were missed.

Exact fix

Add a required Search Receipt section immediately after the executive summary.

Missing target fallback

Medium-4 pts
Evidence

The target system is named, but the prompt does not define what happens if it cannot be found.

Why it matters

Without a fallback rule, the executor may infer a nearby target and produce confident but misplaced work.

Exact fix

Tell the executor to stop and report the missing path instead of inventing a replacement target.

Before searching, identify the active working directory and list the exact roots you will inspect. Do not search unrelated private folders, credentials, environment files, browser profiles, or personal documents.

Under Discovery Method

Constrain semantic search to local sources.

For semantic search, use only local filenames, nearby terminology, local indexes already available in the environment, or manual inspection of likely files. Do not call web or external services.

Revised prompt

A safer runnable version

This sample keeps the prompt close to the original while closing the main execution risks.

Patched prompt excerpt

Review the target workspace in read-only mode and produce a concise context pack with exact file references.

Before searching, identify the active working directory and list the exact roots you will inspect. Do not search unrelated private folders, credentials, environment files, browser profiles, or personal documents.

If the named target cannot be found, stop and report the missing path instead of inferring a replacement target.

Use only local filenames, nearby terminology, local indexes already available in the environment, or manual inspection of likely files. Do not call web or external services.

After the executive summary, include a Search Receipt with working directory, roots searched, roots skipped, search methods used, file types inspected, and inaccessible relevant paths.
